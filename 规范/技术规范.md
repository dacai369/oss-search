# 技术规范

## 1. 流水线总览

```
需求输入 → 意图解析·能力拆解(LLM) → 查询扩展 → 多源并行采集(可插拔适配器)
        → 归并去重·实体对齐 → 质量评级(多维) → 排序推荐 + 选型报告
```

## 2. 技术栈

| 层 | 选型 | 理由 |
|---|---|---|
| 语言 | Python 3.11+ | 爬取 / LLM / 数据生态最全 |
| LLM | 宿主 agent 提供（不绑定 provider） | Skill 跑在谁的环境用谁的模型 |
| 相关度 | BM25 近似（V2 再上本地 embedding） | 不引入 embedding 依赖，可离线 |
| 存储 | 无（纯内存流水线） | 单次运行无需持久化 |
| 入口 | CLI + Skill 封装 | 可脚本化，可被 AI 工具调用 |

## 3. 设计决策

### 意图拆解：结构化 Prompt + 固定 JSON Schema
由宿主 agent 完成拆解，输出符合 `IntentSpec` Schema 的 JSON，引擎只负责校验加载。好处：可测试、可复现、不依赖任何特定 LLM。脱离 agent 单跑时有可选的兜底 LLM 客户端（`llm_client.py`）。

### 信源：内置白名单 + 运行时自定义
内置白名单见 [`信源清单.yaml`](信源清单.yaml)。调用时可通过 `--source URL` 追加自定义信源，报告中会展示本次使用的所有信源。

### LLM 解耦
引擎不绑定任何 LLM provider，不需要任何 LLM key。"动脑"步骤（意图拆解）由宿主 agent 完成；"干活"步骤（采集/评分/渲染）由引擎完成，纯确定性计算，可离线运行。

### 接口层
引擎为独立 Python 库/CLI，可通过 `pip install` 安装或直接 `python -m` 运行。Skill 封装（`SKILL.md`）提供对话触发入口。报告产出 Markdown + 自包含 HTML 两种格式。

## 4. 核心数据结构

### 4.1 IntentSpec（意图拆解输出）
见 [`意图拆解Schema.md`](意图拆解Schema.md)。

### 4.2 Candidate（采集层统一输出）
所有适配器必须输出此结构：

```python
{
  "id": "github:owner/repo",      # 全局唯一，source:identifier
  "source": "github",             # github | npm | pypi
  "name": "repo",
  "full_name": "owner/repo",
  "url": "https://...",
  "description": "...",
  "language": "Python",
  "license": "MIT",
  "stars": 12000,
  "forks": 800,
  "last_commit_at": "2026-05-01T...",
  "open_issues": 30,
  "contributors": 45,            # 可选，拿不到填 null
  "topics": ["search", "llm"],
  "readme_excerpt": "...",
  "matched_capability": "cap-1",  # 命中的能力 id
  "raw": {}                       # 原始 API 返回
}
```

> 拿不到的字段填 `null`，不要伪造。评级层按缺失降权。

### 4.3 ScoredCandidate（评级层输出）
在 Candidate 基础上加：

```python
{
  "scores": {
    "activity": 0.0-1.0,
    "community": 0.0-1.0,
    "maturity": 0.0-1.0,
    "security": 0.0-1.0,
    "license_fit": 0.0-1.0,
    "relevance": 0.0-1.0
  },
  "total_score": 0.0-1.0,
  "grade": "A|B|C|D",
  "reasons": ["活跃度高：近90天120次提交", "..."]
}
```

## 5. 模块接口约定

```python
# 意图拆解
def load_intent_spec(data: dict, original_request: str = "") -> IntentSpec: ...
def parse_intent(request: str) -> IntentSpec: ...  # 兜底，需 LLM

# 采集适配器（统一接口，可插拔）
class SourceAdapter(Protocol):
    name: str
    def search(self, capability: Capability, spec: IntentSpec) -> list[Candidate]: ...

# 归并去重
def dedup(candidates: list[Candidate]) -> list[Candidate]: ...

# 评级
def score(candidates: list[Candidate], spec: IntentSpec) -> list[ScoredCandidate]: ...

# 报告
def build_report(scored: list[ScoredCandidate], spec: IntentSpec) -> str: ...
def build_html(markdown: str) -> str: ...
```

## 6. 质量评级维度与权重

| 维度 | 权重 | 信号 |
|---|---|---|
| relevance 契合度 | 0.30 | BM25 关键词匹配（V2 升级为 embedding 相似度） |
| activity 活跃度 | 0.20 | 近 90 天提交频率、issue 响应 |
| community 社区 | 0.15 | stars / forks / 贡献者数量 |
| maturity 成熟度 | 0.15 | 版本号、文档/测试/License 完整性 |
| license_fit 许可证 | 0.10 | 是否符合用户约束 |
| security 安全 | 0.10 | 已知 CVE（V2 接 OSV，当前中性处理） |

评级：A ≥ 0.80，B ≥ 0.65，C ≥ 0.50，D < 0.50。
