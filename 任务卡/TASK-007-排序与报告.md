## 目标
实现 `build_report(scored, spec) -> Report`：按能力分组，每个能力给 Top-N 候选 + 评级 + 理由 + 风险，最后给整体组合方案建议。同时输出 **Markdown 与自包含 HTML 两种格式**（ADR-007）。

## 实现要点
- 按 capability 分组排序，每组取 Top-N（默认 5，可配）。
- 报告含：需求回显 → 每能力推荐表（项目/grade/分数/理由/license）→ 整体组合建议 + 风险提示。
- 输出 **Markdown**（机器/CLI 友好）+ **单文件 HTML**（样式内联、无外链、可直接发给人看/存档）。
- HTML 由同一份结构化数据渲染，避免两套逻辑分叉。

## 验收标准（DoD）
- [x] 报告按能力分组，每组 Top-N 排序正确
- [x] 每个推荐显示评级、关键理由、license、链接
- [x] 给出整体组合建议与风险（如 license 冲突、维护停滞）
- [x] 同时产出 Markdown + 单文件 HTML（样式内联、可独立打开）
- [x] 用 TASK-001 的样例需求端到端跑出一份可读报告
