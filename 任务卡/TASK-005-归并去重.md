## 目标
实现 `dedup(candidates) -> list[Candidate]`：把指向同一真实项目的多来源结果合并成一条。

## 实现要点
- 归一化键：优先用仓库地址（host+owner+repo 规范化），其次包名→仓库映射。
- 合并策略：保留信息最全的一条，来源记入 `raw.sources`（如 github+npm+content 都提到 → 加权信号，给评级用）。
- 内容源的"提及"映射到真实仓库（配合 TASK-004）。

## 验收标准（DoD）
- [x] 同项目跨源（github + npm + 文章提及）能合并为 1 条
- [x] 大小写/末尾斜杠/.git 后缀等 URL 变体能归一
- [x] 合并后保留"被几个源提及"的计数
- [x] 有覆盖典型重复场景的测试
